## 2026-03-22 - Providing touch feedback and accessibility to custom Flutter buttons
**Learning:** For custom buttons in Flutter, using a `GestureDetector` wrapped around a `Container` doesn't provide native touch ripple feedback and doesn't expose the button semantics properly to screen readers out of the box. Using `Semantics(button: true)` along with `Material(color: Colors.transparent)`, `Ink` and `InkWell` solves both issues while keeping custom designs intact.
**Action:** Prefer `Material` + `InkWell` for custom buttons rather than raw `GestureDetector` + `Container`, and wrap them in `Semantics(button: true)` to ensure screen reader accessibility. Also, when an iOS user needs to type an IP address, `TextInputType.number` doesn't show decimal points, but `TextInputType.numberWithOptions(decimal: true)` does.
