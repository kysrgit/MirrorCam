## 2024-05-23 - Prevent Cross-Site WebSocket Hijacking (CSWH) in local Signaling Server
**Vulnerability:** The local `SignalingServer` running on `8765` was accepting WebSocket upgrade requests from any client. This allows Cross-Site WebSocket Hijacking (CSWH) where a malicious website could open a WebSocket connection to `ws://127.0.0.1:8765/ws` from a user's browser, allowing it to communicate with the native app's local server.
**Learning:** Native apps using `web_socket_channel` or standard Dart `WebSocket` do not send an `Origin` header by default, while web browsers are required by the specification to send the `Origin` header on all cross-origin requests.
**Prevention:** Explicitly check for the `Origin` header in the `HttpRequest` during the upgrade process. Reject the connection with a `403 Forbidden` if an `Origin` header is present, as it indicates the request is originating from a web browser (potentially malicious) rather than our trusted native application.
